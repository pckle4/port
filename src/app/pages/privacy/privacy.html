01 Data Collection Principles

Ansh Shah ("we", "us", "our") operates nowhile.com (the "System") strictly adhering to the principle of data minimization. We only ingest, process, and retain telemetry and user-provided datasets that are absolutely vital for maintaining infrastructure stability, security, and establishing professional contact.

02 Telemetry & Access Logging

When interfacing with the System, our routing architecture automatically aggregates non-personally identifiable connectivity metadata:

  • Network Information: IP address strings, gross geolocational mapping (country/region index) for threat intelligence, and bandwidth heuristics.
  • Device Signatures: Browser user-agent variants, rendering engine type, OS fingerprint, and viewport dimensions for responsive layout optimizations.
  • Traffic Flow: Application routing states, ingress/egress points, and API latency measurements.

03 Secure Data Transmission

Any data payload explicitly transmitted via contact channels (Name, Email Identifiers, Messaging Body) is parsed solely for immediate correspondence. We do not aggregate this data for marketing clusters, nor do we serialize it for long-term algorithmic profiling.

04 Local State Management

The System operates entirely without tracking cookies. It solely utilizes secure localStorage API tokens to persist explicit user interactions (e.g., rendering Theme preferences spanning Dark/Light modes). No personalized state leaves your execution environment.

05 External Service Providers

Architecture Backbones

We leverage enterprise providers strictly for structural integrity. Subprocessors include Vercel (Edge-routing and Serverless execution) and third-party cryptographic services (CAPTCHA endpoints against botnet injections).

These entities maintain SOC2/ISO compliant data governance independently. We distribute no organic user data beyond fundamental TCP/IP routing requirements.

06 Data Sovereignty & Subpoenas

Your information is never commoditized, licensed, or swapped. Access is strictly partitioned. Data release to external jurisdictions only occurs if legally compelled via rigorous, cryptographically verifiable warrants originating from competent statutory bodies.
